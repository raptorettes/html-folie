1 — Introduction

Overview and goals

This single-page layout demonstrates: sticky section headers that remain visible while scrolling, responsive layout for mobile and desktop, and print-specific rules that present the content in two printed columns while keeping sections intact (sections start in a new column).

The live site uses UnoCSS utility classes for spacing, typography and layout. The print stylesheet handles column layout and section-breaking rules so printed output is tidy and readable.

2 — Details & Examples

How it works

Sticky headers are achieved via position: sticky; top: 0;. The header has a subtle background and border so text remains legible when it overlaps content.

For print, we set column-count: 2 on the main container and then ensure each .section has break-inside: avoid-column so sections do not split across columns. In addition we use break-before: column on non-first sections to force each section to start at a new column, which yields clean section boundaries in the printed output.

  • Responsive: single column on small screens, natural stacking on small widths
  • Accessible: reduced motion respected
  • Practical: minimal shadow and border for a "plainish" look

4 — Appendix & Notes

Implementation notes

Implementation tips:

  1. For real projects, compile UnoCSS at build-time to purge unused utilities and enable custom presets.
  2. Adjust @page and margins to match your target paper size (A4, Letter, etc.).
  3. If you want each section to always start on a new page instead of a column, change break-before: column to page-break-before: always or break-before: page inside the print media query.
